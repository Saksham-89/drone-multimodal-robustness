# Project Structure

Full breakdown of every file and directory in this thesis project.

---

## Directory Map

```
thesis/
├── CLAUDE.md           — project brain; read by Claude at the start of every session
├── environment.yml     — conda environment spec (Python 3.10, PyTorch, imagecorruptions, mmdet)
├── docs/               — all documentation and reference materials
├── data/               — dataset storage
├── models/             — cloned model repos
├── src/                — reusable library code (corruption pipeline, evaluation, inference wrappers)
├── experiments/        — top-level scripts that run the 3 experiment blocks
├── results/            — JSON outputs from every inference run
├── analysis/           — notebooks for making tables and figures
└── hpc/                — SLURM job scripts for the EEMCS HPC cluster
```

---

## 1. CLAUDE.md — the project brain

Read automatically by Claude at the start of every conversation. Contains:
- Both research questions (RQ1 = degradation profiling, RQ2 = modality attribution)
- Dataset spec (DroneVehicle, 28,439 pairs, 5 classes, oriented boxes)
- All 3 model architectures and how they fuse
- Exact corruption parameters from Table 2 of the proposal
- The full 210-run experiment scope
- Critical implementation rules (never use COCO eval, zeroing = all-zeros tensor, etc.)

---

## 2. data/

```
data/
├── DroneVehicle/
│   ├── train/          — training split (not touched during experiments)
│   ├── val/            — validation split
│   └── test/           — test split (only this is used for evaluation)
└── corrupted/          — generated by the corruption pipeline at experiment time
```

Raw dataset goes into `DroneVehicle/` after download. The `corrupted/` folder is populated on-the-fly during inference — corruptions are applied per image rather than pre-generating all 23 × N images to disk.

---

## 3. models/

```
models/
├── ua_cmddet/          — clone of DroneVehicle repo (Sun et al., TCSVT 2022)
├── early_fusion/       — our implementation (concat before encoder, ResNet-50)
└── c2former/           — clone of C2Former repo (Yuan & Wei, IEEE TGRS 2024)
```

These three repos are the blocker for everything else. Once cloned, the inference wrappers in `src/inference/` get wired up.

---

## 4. src/ — library code

Reusable modules. Experiment scripts import from here.

### src/corruption/

| File | What it does |
|---|---|
| `params.py` | All Table 2 severity parameters hardcoded. Auto-generates `ALL_CONDITIONS` — 23 `(modality, corruption_type, severity)` tuples that every experiment loops over. |
| `pipeline.py` | `apply_corruption(image, modality, corruption_type, severity)` — maps thesis corruption names to `imagecorruptions` library calls. Complete dropout returns `np.zeros_like(image)`. Also has `zero_modality()` for Experiment 2. |

### src/evaluation/

| File | What it does |
|---|---|
| `metrics.py` | `compute_ra(corrupted_map, clean_map)` = `corrupted_map / clean_map`. Also `load_map_results()` and `build_ra_table()`. |
| `dota_eval.py` | Stub wrapper around the DOTA devkit. Wire up after cloning DOTA devkit. Required — standard COCO mAP gives wrong results on oriented bounding boxes. |

### src/inference/

| File | What it does |
|---|---|
| `base.py` | Abstract `BaseInferenceRunner` with `load_model()` and `run(rgb, tir)` interface. Provides `run_rgb_only()` and `run_tir_only()` for free — they call `run()` with one stream zeroed. |
| `ua_cmddet.py` | Stub — fill in after cloning UA-CMDet repo. |
| `c2former.py` | Stub — fill in after cloning C2Former repo. |
| `early_fusion.py` | Stub — fill in after implementing the early fusion architecture. |

The abstract base class means all experiment scripts are model-agnostic — they call the same interface regardless of which model is running.

---

## 5. experiments/

| File | What it runs |
|---|---|
| `exp0_baseline.py` | Experiment 0: clean inference on all 3 models. Validates mAP matches published figures for UA-CMDet and C2Former. Hard gate before any corruption work. |
| `exp1_corruption.py` | Experiment 1 (RQ1): loops 3 models × 23 corruption conditions = 69 runs. Applies corruption to the relevant modality, runs full dual-modality inference, computes RA. |
| `exp2_modality_removal.py` | Experiment 2 (RQ2): for each of the 23 conditions, runs RGB-only and TIR-only inference = 138 additional runs. |

Each run saves a separate JSON to `results/`. Skip-if-exists logic means HPC jobs are resumable if they die mid-way.

### experiments/configs/

Three YAML files (`ua_cmddet.yaml`, `c2former.yaml`, `early_fusion.yaml`) with paths to checkpoints, data directories, and eval settings. Update checkpoint paths after training.

---

## 6. results/

```
results/
├── exp0_baseline/           — one JSON per model: {model, map}
├── exp1_corruption/         — one JSON per (model, modality, corruption, severity)
└── exp2_modality_removal/   — one JSON per (model, modality, corruption, severity, config)
```

Example filename: `ua_cmddet__rgb__gaussian_noise__s1.json`  
Example content: `{"model": "ua_cmddet", "modality": "rgb", "corruption": "gaussian_noise", "severity": 1, "map": 0.412, "ra": 0.887}`

## 7. analysis/

```
analysis/
├── notebooks/
│   ├── rq1_degradation_profiles.ipynb   — RA tables and degradation curves per model/corruption
│   └── rq2_modality_priority.ipynb      — modality priority maps across corruption conditions
```

These notebooks load from `results/` and generate all tables and figures. To be built after experiments complete.

## 8. hpc/

SLURM job scripts for the EEMCS HPC cluster. Update `partition`, `--gres`, and `source activate <env>` to match cluster config before submitting.

| Script | Purpose |
|---|---|
| `train_ua_cmddet.sh` | Train UA-CMDet on DroneVehicle train split |
| `train_c2former.sh` | Train C2Former on DroneVehicle train split |
| `train_early_fusion.sh` | Train early fusion model |
| `run_exp1.sh` | Submit Experiment 1 (69 inference runs) |
| `run_exp2.sh` | Submit Experiment 2 (138 inference runs) |
